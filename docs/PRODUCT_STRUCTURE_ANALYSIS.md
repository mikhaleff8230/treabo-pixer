# Анализ структуры товаров в базе данных

## 📊 Общая структура

В системе используются **ТРИ РАЗНЫЕ ТАБЛИЦЫ** для товаров:

1. **`products`** - Обычные товары (старая система)
2. **`product_groups`** - Группы товаров (новая система)
3. **`product_skus`** - SKU/вариации товаров (новая система)

## 🔍 Детальное сравнение

### 1. Таблица `products` (Обычные товары)

**Основные поля:**
- `id`, `name`, `slug`
- `description`, `type_id`
- `price`, `sale_price`, `sku`
- `quantity`, `in_stock`
- `height`, `width`, `length`
- `image`, `gallery`
- `status`, `product_type` (SIMPLE/VARIABLE)
- `unit`, `is_taxable`

**Особенности:**
- ✅ Самостоятельная таблица
- ✅ Может быть SIMPLE (простой) или VARIABLE (вариативный)
- ✅ Все данные хранятся в одной записи

### 2. Таблица `product_groups` (Группы товаров)

**Основные поля:**
- `id`, `title`, `slug`
- `description`, `short_description`
- `category_id`, `type_id`, `shop_id`
- `brand_id`, `brand_type` (manufacturer/author)
- `main_image`, `gallery`, `video`
- `height`, `length`, `width`, `weight`
- `status`, `language`
- `meta` (JSON для дополнительных полей)

**Особенности:**
- ✅ Содержит общую информацию о группе товаров
- ✅ Связана с категорией, типом, магазином
- ✅ Содержит габариты и вес (общие для всех SKU)
- ✅ Имеет связь `hasMany` с `product_skus`

### 3. Таблица `product_skus` (SKU/Вариации)

**Основные поля:**
- `id`, `group_id` (FK → product_groups)
- `sku`, `slug`, `title`
- `price`, `old_price`
- `quantity`, `barcode`
- `image` (JSON)
- `is_active`, `is_digital`, `is_disable`
- `description`
- `language`, `meta` (JSON)

**Связи:**
- `belongsTo` ProductGroup (через `group_id`)
- `belongsToMany` AttributeValue (через `product_sku_property_values`)

**Особенности:**
- ✅ Каждый SKU - это отдельная запись в базе
- ✅ Наследует общую информацию из `product_groups` через связь
- ✅ Имеет свои уникальные: цену, количество, артикул, изображение
- ✅ Связан с атрибутами через pivot таблицу

## 🔗 Связи между таблицами

```
ProductGroup (1) ──hasMany──> (N) ProductSku
ProductSku (N) ──belongsTo──> (1) ProductGroup

ProductSku (N) ──belongsToMany──> (N) AttributeValue
  через pivot: product_sku_property_values
```

## ❓ Ответ на вопрос

**Вопрос:** "Получается у нас каждое SKU получается как отдельный товар просто часть полей наследует из настроек группы?"

**Ответ:** ✅ **ДА, ВЕРНО!**

### Как это работает:

1. **ProductGroup** хранит общую информацию:
   - Название группы, описание
   - Категория, тип, магазин
   - Габариты (общие для всех SKU)
   - Статус публикации

2. **ProductSku** - это отдельные товары:
   - Каждый SKU = отдельная запись в таблице `product_skus`
   - Имеет свой `id`, `slug`, `sku`
   - Имеет свою цену, количество, изображение
   - Наследует общую информацию через связь `group_id`

3. **Наследование полей:**
   - ❌ **НЕ через SQL JOIN** (не автоматически)
   - ✅ **Через Eloquent отношения** (`$sku->group->title`)
   - ✅ **Через код** - при отображении данные берутся из обеих таблиц

## 📋 Что наследуется из группы в SKU

### Через связь `group()`:
- `title` → `$sku->group->title`
- `description` → `$sku->group->description`
- `category_id` → `$sku->group->category_id`
- `type_id` → `$sku->group->type_id`
- `shop_id` → `$sku->group->shop_id`
- `height`, `length`, `width`, `weight` → `$sku->group->height`
- `status` → `$sku->group->status`

### Собственные поля SKU:
- `title` - название конкретного SKU
- `price` - цена конкретного SKU
- `quantity` - количество конкретного SKU
- `sku` - артикул конкретного SKU
- `image` - изображение конкретного SKU
- `is_active` - активность конкретного SKU

## ⚠️ Важные моменты

1. **НЕТ единого формата:**
   - `products` и `product_skus` - это разные таблицы
   - Разные структуры полей
   - Разные модели Eloquent

2. **SKU НЕ наследует автоматически:**
   - Данные из группы нужно получать через связь
   - При сохранении SKU не копируются данные группы
   - При изменении группы нужно обновлять все SKU вручную (если нужно)

3. **Рекомендации:**
   - ✅ Использовать Eloquent отношения для получения данных группы
   - ✅ При отображении SKU загружать группу через `with('group')`
   - ✅ При создании SKU проверять наличие группы
   - ✅ При удалении группы удаляются все SKU (cascade)

## 🔄 Сравнение с обычными товарами

| Параметр | Product (старая) | ProductGroup + ProductSku (новая) |
|----------|------------------|-----------------------------------|
| Структура | Одна таблица | Две связанные таблицы |
| Вариации | В одной записи (VARIABLE) | Отдельные записи (SKU) |
| Наследование | Нет | Через связи Eloquent |
| Гибкость | Ограничена | Высокая |
| Производительность | Быстрее (одна таблица) | Медленнее (JOIN) |

## ✅ Вывод

**Да, каждое SKU - это отдельный товар в базе данных**, который:
- Имеет свою запись в таблице `product_skus`
- Связан с группой через `group_id`
- Получает общую информацию через Eloquent отношения
- Имеет свои уникальные поля (цена, количество, изображение)

**Это НЕ единый формат** - это две разные системы:
- Старая: `products` (один товар = одна запись)
- Новая: `product_groups` + `product_skus` (один товар = группа + несколько SKU)

## 🌐 Унификация URL на фронтенде

### ✅ Реализовано: Единый формат URL с использованием `/element/`

**На фронтенде все товары используют единый формат URL:**
```
/element/{slug}
```

### Как это работает:

1. **Роут `/element/{slug}` на бэкенде:**
   - Ищет товар в таблице `products` (обычные товары)
   - Если не найден, ищет в таблице `product_skus` (SKU)
   - Если не найден, ищет в таблице `product_groups` (группы)
   - Возвращает товар в едином формате JSON

2. **Приоритет поиска:**
   ```
   1. Product (обычные товары)
   2. ProductSku (SKU/вариации) ← приоритет для вариаций
   3. ProductGroup (группы товаров)
   ```

3. **Фронтенд получает:**
   - Единый формат ответа
   - Одинаковую структуру данных
   - Возможность обрабатывать все типы товаров одинаково

4. **Страница `/element/[slug].tsx` на фронтенде:**
   - Универсальная страница для всех типов товаров
   - Автоматически определяет тип товара (Product/ProductSku)
   - Отображает данные в зависимости от типа
   - Для SKU использует данные из группы при необходимости

### Преимущества:

✅ **Единый формат URL** - `/element/{slug}` для всех типов товаров
✅ **SEO-дружелюбные URL** - короткие и понятные
✅ **Обратная совместимость** - старые URL `/products/{slug}` редиректятся на `/element/{slug}` (301)
✅ **Унифицированная обработка** - один компонент на фронтенде для всех типов
✅ **Замена product на element** - единообразное именование для всех товаров

### Примеры URL:

```
/element/krasivaya-kartina          ← Обычный товар (Product)
/element/nizhnee-bele-seksi-meksi-chernyy-xs  ← SKU (ProductSku)
/element/gruppa-tovarov            ← Группа (ProductGroup)
```

### Редиректы:

- `/products/{slug}` → `/element/{slug}` (301 редирект для SEO)
- Старые ссылки автоматически перенаправляются на новый формат

Все эти URL обрабатываются через один роут `/element/{slug}` и возвращают товар в едином формате!
