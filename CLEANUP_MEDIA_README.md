# 🧹 Очистка медиа-файлов

## Проблема
После удаления товаров и мест изображения остаются в S3 и локальном хранилище, занимая место и увеличивая расходы на хранение.

## Решение
Созданы скрипты для анализа и безопасной очистки неиспользуемых медиа-файлов.

## 📋 Скрипты

### 1. `verify-unused-files.php` - Проверка безопасности ⭐
```bash
cd /var/www/sancan.ru/pixer-api
php verify-unused-files.php
```

**ОБЯЗАТЕЛЬНО запустите сначала!**
- Детальный анализ файлов перед удалением
- Показывает примеры файлов для удаления
- Проверяет безопасность путей
- Показывает статистику по папкам

### 2. `analyze-s3-usage.php` - Анализ использования S3
```bash
cd /var/www/sancan.ru/pixer-api
php analyze-s3-usage.php
```

**Что показывает:**
- Общее количество файлов в S3
- Распределение по папкам (products/images, places/videos и т.д.)
- Распределение по типам файлов (jpg, png, webp и т.д.)
- Количество используемых и неиспользуемых файлов
- Размер неиспользуемых файлов

### 3. `cleanup-local-storage.php` - Очистка локального хранилища
```bash
cd /var/www/sancan.ru/pixer-api
php cleanup-local-storage.php
```

**Что делает:**
- Анализирует папки storage/
- Находит неиспользуемые медиа-файлы
- Очищает старые логи и кэш
- **Двойная проверка безопасности**

### 4. `quick-cleanup.php` - Быстрая очистка S3
```bash
cd /var/www/sancan.ru/pixer-api
php quick-cleanup.php
```

**Что делает:**
- Находит все неиспользуемые файлы в S3
- Показывает примеры
- Предлагает удалить их
- Удаляет файлы после подтверждения

### 5. `cleanup-orphaned-media.php` - Полная очистка S3
```bash
cd /var/www/sancan.ru/pixer-api
php cleanup-orphaned-media.php
```

**Возможности:**
- Детальный анализ файлов
- Создание резервной копии
- Безопасное удаление
- Подробная статистика

## 🚀 Рекомендуемый порядок действий

### Шаг 1: Проверка безопасности (ОБЯЗАТЕЛЬНО!)
```bash
php verify-unused-files.php
```
**Этот скрипт покажет:**
- Какие файлы будут удалены
- Примеры файлов для удаления
- Проверку безопасности путей
- Статистику по папкам

### Шаг 2: Анализ S3
```bash
php analyze-s3-usage.php
```

### Шаг 3: Очистка локального хранилища
```bash
php cleanup-local-storage.php
```

### Шаг 4: Очистка S3
```bash
# Быстрая очистка
php quick-cleanup.php

# Или полная очистка с резервными копиями
php cleanup-orphaned-media.php
```

## ⚠️ КРИТИЧЕСКИ ВАЖНО

### Перед удалением ОБЯЗАТЕЛЬНО:
1. **Запустите `verify-unused-files.php`** - он покажет детальную информацию
2. **Проверьте примеры файлов** - убедитесь, что они действительно не нужны
3. **Сделайте резервную копию** - на всякий случай
4. **Проверьте, что товары работают** - убедитесь, что изображения отображаются

### Система безопасности:
- **Двойное подтверждение** - нужно ввести "yes" и "DELETE"
- **Проверка путей** - файлы вне storage/ будут пропущены
- **Проверка существования** - файлы проверяются в БД перед удалением
- **Детальное логирование** - все действия записываются

### Резервная копия:
```bash
# Создайте резервную копию S3
aws s3 sync s3://your-bucket-name ./backup-s3-$(date +%Y%m%d)/
```

## Автоматическая очистка

### Добавьте в crontab для регулярной очистки:
```bash
# Очистка каждую неделю в воскресенье в 2:00
0 2 * * 0 cd /var/www/sancan.ru/pixer-api && php quick-cleanup.php >> storage/logs/cleanup.log 2>&1
```

## Мониторинг

### Проверяйте логи:
```bash
tail -f storage/logs/laravel.log | grep "deleted from S3"
```

### Регулярно анализируйте использование:
```bash
php analyze-s3-usage.php
```

## Результат

После очистки:
- ✅ Освободится место в S3
- ✅ Снизятся расходы на хранение
- ✅ Ускорится работа с файлами
- ✅ Упростится управление медиа

## Поддержка

При возникновении проблем:
1. Проверьте логи: `tail -f storage/logs/laravel.log`
2. Убедитесь, что S3 настроен правильно
3. Проверьте права доступа к файлам
4. Создайте резервную копию перед очисткой
